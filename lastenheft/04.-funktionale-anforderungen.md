# 4. Funktionale Anforderungen

Hinweis: Die Wichtigkeit der einzelnen Anforderungen (\<muss>|\<sollte>|\<wird>) wurden von den Interviewteams situativ nach den Formulierungen der Interviewpartner entschieden.

## 4.1 Zustände
**FU-0101** Das System muss Wartungsingenieuren die Möglichkeit bieten Alarmzustände zu definieren [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0102** Das System muss Wartungsingenieuren die Möglichkeit bieten Alarmreaktionen zu Alarmzuständen zuzuordnen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 

**FU-0103** Falls ein Alarmzustand eintritt, muss das System fähig sein die zugeordnete Alarmreaktionen auszuführen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 

**FU-0104** Das System muss fähig sein, Alarmzustände zu erkennen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 

**FU-0105** Das System sollte Wartungsingenieuren die Möglichkeit bieten, eine manuelle Auswertung des Maschinenzustands durchzuführen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 




## 4.2 Schnittstelle
**FU-0201** Das System muss über eine eigene Schnittstelle die Ingestion via Batch unterstützten [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)]. 

**FU-0202** Das System muss über eine eigene Schnittstelle die Ingestion via Datenstreaming unterstützen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101), [QU-0202](../lastenheft/09.-quellen.md#QU-0202)].

**FU-0203** Das System sollte Schnittstellen für verschiedene Maschinen oder Sensoren bereitstellen [[QU-0105](../lastenheft/09.-quellen.md#QU-0105)]. 

**FU-0204** Das System muss eine Schnittstelle für den Datenimport und Export bereitstellen [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)]. 

**FU-0205** Das System muss Software-Systeme mit ihren Schnittstellen einbinden [[QU-0106](../lastenheft/09.-quellen.md#QU-0106)].




## 4.3 Algorithmen und KI-Modelle
**FU-0301** Das System sollte Algorithmen zur Anomalieerkennung bereitstellen [[QU-0105](../lastenheft/09.-quellen.md#QU-0105), [QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0302** Das System sollte benötigten oder trivialen Daten filtern (Anomalieerkennung) [[QU-0105](../lastenheft/09.-quellen.md#QU-0105)]. 

**FU-0303** Das System sollte KI – Modelle integrieren können [[QU-0105](../lastenheft/09.-quellen.md#QU-0105)]. 

**FU-0304** Das System sollte KI-Modelle trainieren beziehungsweise verbessern [[QU-0105](../lastenheft/09.-quellen.md#QU-0105)]. 

**FU-0305** Das System muss Plattformentwicklern die Möglichkeit bieten selbstgeschriebene Auswertungssoftware anzubinden [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 

**FU-0306** Das System sollte eine Anbindung zu Process Mining und Business Process Automation Lösungen erlauben [[QU-0203](../lastenheft/09.-quellen.md#QU-0203)]. 

**FU-0307** Das System sollte Variable Software zur Analyse großer Datenmengen, komfortable Tools zum Austausch/zur Transformation von Daten zwischen verschiedenen Plattformen / Programmier-Paradigmen, z.B. relationale DB, R, Java, Python, TensorFlow/Keras. Hardware-Unterstützung bei der Analyse größerer Datenmengen, z.B. GPU-Server etc. bieten [[QU-0203](../lastenheft/09.-quellen.md#QU-0203)]. 

**FU-0308** Die Data Platform sollte variable Software zur Analyse größerer Datenmengen anbieten [[QU-0202](../lastenheft/09.-quellen.md#QU-0202)]. 

**FU-0309** Das System sollte Plattformentwicklern die Möglichkeit bieten unternehmensspezifische Modifikationen vorzunehmen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 



## 4.4 Wartung
**FU-0401** Das System muss in der Lage sein, aufgrund von bereitgestellten Maschinendaten entsprechende Wartungstermine zu errechnen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 


**FU-0402** Das System sollte die Wartungsdaten von Anlagen erfassen und auswerten, damit diese in den Wartungskonzept einfließen können [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0403** Das System sollte den Zeitpunkt, zu dem die Wartung des Geräts erfolgen soll, bestimmen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0404** Das System muss Wartungsingenieuren die Möglichkeit bieten Alarmreaktionen zu definieren [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)]. 



## 4.5 Mitarbeiter
**FU-0501** Das System muss den Mitarbeitern im Logistikleitstand die Möglichkeit bieten die Stärken und Schwächen eines Kommissionierers zu bestimmen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0502** Das System muss dem Lagerleiter die Möglichkeit bieten, die Schichtzeiten der Mitarbeiter zu organisieren [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0503** Das System muss den Mitarbeitern im Logistikleitstand die Möglichkeit bieten alle Personen, Gabelstapler und andere Elemente im Logistikzentrum nachzuverfolgen und deren Positionsdaten zu betrachten [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 




## 4.6 Störungen
**FU-0601** Das System muss die aufgetretenen Störungen an Geräten erfassen und auswerten [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0602** Das System muss bestimmen, wie oft eine Störung aufgetreten ist und wievielmal diese Störung aus demselben Grund entstanden ist [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0603** Das System muss den Mitarbeitern im Logistikleitstand die Möglichkeit bieten Störungsdaten für die Fehleranalyse aufzurufen und diese zu analysieren [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 




## 4.7 Aufträge
**FU-0701** Das System muss das Abarbeiten von Aufträgen optimieren, indem es beispielsweise Aufträge vorzieht oder die Reihenfolge der Aufträge verändert [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0702** Das System sollte die Anzahl der benötigten Kommissionierer für das Abwickeln von Auftragspositionen bestimmen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 

**FU-0703** Das System soll den Anwender in verschiedenen Entscheidungsprozessen unterstützen [[QU-0106](../lastenheft/09.-quellen.md#QU-0106)]. 

**FU-0704** Das System muss Auskunft über die Reihenfolge der Abarbeitung der Aufträge geben [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)]. 



## 4.8 Daten
**FU-0801** Das System muss in die Lage sein, die von Unternehmen bereitgestellte Daten einem Mandanten zuzuordnen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)]. 

**FU-0802** Das System muss berechtigten Nutzern den Zugriff auf die von Unternehmen bereitgestellten Daten ermöglichen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0803** Das System muss in der Lage sein Daten zur gemeinschaftlichen Nutzung und zum Austausch bereitzustellen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)]. 

**FU-0804** Das System muss berechtigten Nutzern ermöglichen die vorher bereitgestellten Daten zu anonymisieren [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0805** Das System muss berechtigten Nutzern die Möglichkeit geben, gespeicherte Daten zu transformieren und zu kopieren [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0806** Das System muss Daten in strukturierter und unstrukturierter Form speichern können [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0807** Das System muss Metadaten zu gespeicherten Daten führen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0808** Das System muss Daten nach einem Datenbewirtschaftungskonzept führen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0809** Das System muss berechtigen Mandanten die Möglichkeit geben, ihre Daten miteinander zu teilen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0810** Das System muss den Austausch von gespeicherten Daten zwischen Mandanten ermöglichen [[QU-0101](../lastenheft/09.-quellen.md#QU-0101)].

**FU-0811** Das System muss personenbezogene Daten, anonymisiert oder pseudonymisiert verwalten [[QU-0105](../lastenheft/09.-quellen.md#QU-0105)].

**FU-0812** Das System muss die in den Logistikzentren erhobenen Sensordaten aggregieren [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0813** Das System muss durch die Kombination der Lichtschrankendaten Auskunft über Fehlfunktionen geben [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0814** Das System muss die Laufzeit der Förderer erfassen und auswerten [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0815** Das System muss Daten aus mehreren Logistikzentren holen und diese auswerten [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0816** Das System muss den Mitarbeitern im Logistikleitstand die Möglichkeit bieten, Daten von mehreren Logistikzentren zu aggregieren [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0817** Das System muss die Möglichkeit bieten, Unregelmäßigkeiten in erhaltenen Daten zu erkennen [[QU-0108](../lastenheft/09.-quellen.md#QU-0108)].

**FU-0818** Das System muss die Möglichkeit bieten, die Lebensdauer von eingesetzten Produkten zu bestimmen [[QU-0108](../lastenheft/09.-quellen.md#QU-0108)].

**FU-0819** Das System sollte fähig sein, Sensordaten mit einer Frequenz von 8-15ms entgegennehmen zu können [[QU-0108](../lastenheft/09.-quellen.md#QU-0108)].

**FU-0820** Das System muss diverse schweißtechnische Daten zusammenführen [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0821** Nachdem schweißtechnische Daten zusammengeführt worden sind, muss das System die Qualität der Schweißung bewertbar darstellen [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0822** Nachdem schweißtechnische Daten zusammengeführt worden sind, muss das System die Elektrodenstandzeit bewertbar darstellen [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0823** Das System sollte wichtige von unwichtigen Daten trennen [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0824** Das System sollte gleichartige Daten von verschiedenen Unternehmen zusammenzuführen und verarbeiten [[QU-0107](../lastenheft/09.-quellen.md#QU-0107)].

**FU-0825** Das System muss fähig sein die Rohdaten angebundener verschiedenartiger Sensoren zu speichern (z.B. Temperaturen, Dehnungsmesstreifen, Beschleunigungssensoren, Motorströme, Leistungsbedarf) [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0826**  Das System sollte Plattformentwicklern die Möglichkeit bieten ein Standardformat für Sensordaten festzulegen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0827** Das System sollte Plattformentwicklern die Möglichkeit bieten unternehmensspezifische Modifikationen vorzunehmen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0828** Das System muss Daten (Sensordaten, Standardformate etc) auf ihren Ursprung zurückführen können [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0829** Das System muss Daten (Sensordaten, Standardformate etc) auf ihren Zeitpunkt zurückführen können [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0830** Das System muss Daten über einen längeren Zeitraum von mindestens 2 Jahren nachhalten [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0831** Das System muss fähig sein Datenquellen mit unterschiedlichen Datenabtastraten zu unterstützen [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0832** Das System soll verschiedene erzeugte Prozessdaten aus den Produzierenden Maschinen speichern, welche für unterschiedliche Szenarien genutzt werden können [[QU-0106](../lastenheft/09.-quellen.md#QU-0106)].

**FU-0833** Das System soll die von Maschinen erzeugten Prozessdaten auswerten können [[QU-0106](../lastenheft/09.-quellen.md#QU-0106)]. 

**FU-0834** Das System sollte die hochgeladenen Daten automatisiert aufbereiten können (Verknüpfung und Analyse) [[QU-0201](../lastenheft/09.-quellen.md#QU-0201)].

**FU-0835** Das System sollte die Vernetzung von Maschinen und Prozessketten und Verarbeitung von Daten verwirklichen [[QU-0203](../lastenheft/09.-quellen.md#QU-0203)].

**FU-0836** Die Data Platform sollte Forschenden den Zugriff auf eine Sammlung fundierter und realistischer Daten und Sensorwerte ermöglichen [[QU-0202](../lastenheft/09.-quellen.md#QU-0202)].

**FU-0837** Das System sollte Trainingsdaten gelabelled/ungelabelled für supervised/unsupervised learning aus unterschiedlichen Domänen speichern [[QU-0203](../lastenheft/09.-quellen.md#QU-0203)].


## 4.9 Anzeige
**FU-0901** Das System muss Zustände von Produktionsmaschinen darstellen können [[QU-0101](../lastenheft/09.-quellen.md#QU-0101), [QU-0201](../lastenheft/09.-quellen.md#QU-0201)].

**FU-0902** Das System muss Plattformentwicklern die Möglichkeit bieten Dashboards für eine Live-Anzeige des Maschinenzustands anzubinden [[QU-0102](../lastenheft/09.-quellen.md#QU-0102)].

**FU-0903** Das System muss die besetzten Kommissionierplätze dem Lagerleiter anzeigen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0904** Das System muss den Mitarbeitern im Logistikleitstand die Materialflüsse im Lager anzeigen [[QU-0104](../lastenheft/09.-quellen.md#QU-0104)].

**FU-0905** Das System muss die hochgeladenen Daten visuell aufbereitet darstellen können [[QU-0201](../lastenheft/09.-quellen.md#QU-0201)].

**FU-0906** Das System sollte Projekte und deren Ergebnisse in verständlicher Form anzeigen [[QU-0203](../lastenheft/09.-quellen.md#QU-0203)].
